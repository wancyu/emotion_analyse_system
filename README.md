# 多模态情感分析系统

## 项目概述

7A 组 — 社交媒体场景（微博 / 小红书图文）

整合 **视觉（人脸表情）+ 文本（中文情感）** 两种模态，通过多策略融合判断用户真实情感。
重点解决单模态无法处理的 **矛盾场景**：笑脸配讽刺文字、愤怒脸配赞美文字等。

## 核心功能

### 1. 视觉情感识别（7 分类）
- 模型：ResNet18，在 RAF-DB 数据集上训练
- 类别：高兴 / 生气 / 惊讶 / 悲伤 / 恐惧 / 厌恶 / 中性
- 人脸检测：YuNet (OpenCV DNN)
- 输出 512 维特征向量

### 2. 文本情感识别（3 分类）
- 模型：chinese-roberta-wwm-ext（哈工大讯飞）
- 类别：正向 / 负向 / 中性
- 输出 768 维语义向量

### 3. 多模态融合（3 种独立模型，各有训练权重）

| 策略 | 原理 | 门控系数 |
|------|------|---------|
| 早期融合 (Early) | CV 512d + NLP 768d 拼接 → MLP 分类 | — |
| 晚期融合 (Late) | CV/NLP 各一个专家 → 加权投票 | 固定 0.5:0.5 |
| 门控注意力 (Gated) | 学 α 动态分配权重 → 加权拼接 → 分类 | 每样本动态计算 |

融合后输出 5 分类：喜 / 怒 / 哀 / 惊 / 讽。
UI 可通过下拉框实时切换三种策略对比效果。
同时内置冲突检测逻辑：人脸和文字情绪矛盾时自动预警。

### 4. 情绪强度评估（扩展挑战）
- 基于融合置信度映射 1-10 分强度
- 前端实时显示强度进度条

### 5. 性能对比

| 模型 | 准确率 | 讽刺类 F1 |
|------|--------|----------|
| 纯 CV | ~73% | — |
| 纯 NLP | ~68% | — |
| 早期融合 | ~76% | 0.31 |
| 晚期融合 | ~77% | 0.38 |
| 门控注意力 | ~81% | 0.52 |

多模态融合均优于单模态，门控注意力在讽刺类提升最显著。

### 6. 矛盾样本分析典型 case

| 人脸 | 文本 | CV | NLP | 融合 | 门控 α |
|------|------|----|-----|------|-------|
| 笑脸 😊 | "太糟糕了气死我了" | happy 95% | negative 99% | **angry** 100% | 1.00 |
| 愤怒 😡 | "你太好了爱你" | angry | positive | **sarcasm** | 0.85 |
| 笑脸 😊 | "哈哈哈太搞笑了" | happy 95% | positive 99% | **happy** 100% | 0.52 |

矛盾场景下门控 α 会偏向更可信的模态，一致场景下两模态均衡。

### 7. 实时 Demo（扩展挑战 +10 分）
- 摄像头捕捉人脸 → 实时输出 7 种表情概率
- Web 界面：Gradio 搭建，支持上传图片 + 输入文本 + 历史记录

## ⚠️ GitHub 仓库说明

本仓库为课程作业提交用途。原始打包版项目包含内置 Python 运行时（`runtime/`，约 2 GB）和便携启动器（`启动.exe`），但受限于 GitHub 的限制无法直接上传：

| 限制 | 说明 |
|---|---|
| **单文件 100 MB 上限** | `model.safetensors`（391 MB）超出限制，改用 Git LFS 存储 |
| **仓库总容量推荐 ≤ 1 GB** | 仅 `runtime/` 便携 Python 环境就达 2 GB，无法上传 |
| **二进制文件不宜版本管理** | `.exe`、运行时 DLL 等不适合 Git 追踪 |

因此仓库中移除了 `启动.exe` 和 `runtime/`，只保留源代码、模型权重和依赖清单。**课程报告中所述"双击 exe 运行"为本地打包版的使用方式。**

## 本地运行

> 如果你 clone 了本仓库，按以下步骤运行：

```bash
# 1. 安装依赖（仅首次）
pip install -r requirements.txt

# 2. 确保安装 Git LFS（否则模型文件无法正确下载）
git lfs install
git lfs pull

# 3. 启动系统
cd project
python app/app.py
```

浏览器将自动打开 http://127.0.0.1:7860，等待模型加载约 30 秒即可使用。

> 💡 如需还原为"双击 exe"的打包版，可将本仓库 `project/` 目录覆盖到原始打包版对应路径，配合 `runtime/` 和 `启动.exe` 使用。

## 技术栈

| 模块 | 技术 |
|------|------|
| 视觉 | ResNet18 + RAF-DB + OpenCV |
| 文本 | chinese-roberta-wwm-ext + HuggingFace Transformers |
| 融合 | PyTorch + 门控注意力 + 冲突感知 |
| 界面 | Gradio 6.x |
| 数据 | 自建 400 条中文社交媒体图文对（含讽刺标注） |

## 文件结构

```
启动.exe          ← 双击启动
project/          ← 项目代码和模型权重
runtime/          ← 便携 Python 环境
user_data/        ← 历史记录和上传图片（运行时生成）
```

## 常见问题

**点了 exe 没反应？** 检查杀毒软件是否拦截，添加信任即可。

**提示缺少 DLL？** 安装 VC++ 运行时，Windows 11 通常已自带。

**端口被占用？** 关闭占用 7860 端口的程序，或修改 `project/app/app.py` 中的端口号。
